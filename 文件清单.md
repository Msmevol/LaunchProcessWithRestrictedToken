### ✏️ 修改阶段

#### 打开文件
```
文件: Sandboxie\core\drv\verify.c
位置: 第530行左右
函数: KphValidateCertificate()
```

#### 添加代码
在 `Verify_CertInfo.State = 0;` 之后添加：

```c
// 🔓 解除所有限制
Verify_CertInfo.type = eCertEternal;
Verify_CertInfo.level = eCertMaxLevel;
Verify_CertInfo.opt_sec = 1;
Verify_CertInfo.opt_enc = 1;
Verify_CertInfo.opt_net = 1;
Verify_CertInfo.opt_desk = 1;
Verify_CertInfo.expired = 0;
Verify_CertInfo.outdated = 0;
Verify_CertInfo.active = 1;
Verify_CertInfo.grace_period = 0;
Verify_CertInfo.locked = 0;
Verify_CertInfo.lock_req = 0;
Verify_CertInfo.expirers_in_sec = -1;
return STATUS_SUCCESS;
```

💡 **提示**: 详细修改说明请查看 `修改指南.md`

### 🔨 编译阶段

#### 方法A：使用自动化脚本（推荐）

```batch
# 在 Sandboxie 目录下双击运行
.\build_and_sign.bat
```

脚本会自动完成：
1. ✅ 检查编译环境
2. ✅ 编译 LowLevel (Win32)
3. ✅ 编译主项目 (x64)
4. ✅ 检查/启用测试模式
5. ✅ 创建测试证书
6. ✅ 签名所有文件
7. ✅ 创建发布包

**输出**: `Release_Package_YYYYMMDDHHMMSS` 文件夹

#### 方法B：手动编译

```
1. 打开 Visual Studio 2019
2. 打开 Sandbox.sln
3. 配置: Release, 平台: x64
4. 先编译 LowLevel (Win32)
5. 再编译整个解决方案 (Ctrl+Shift+B)
```

💡 **提示**: 详细步骤请查看 `编译和签名指南.md`

### 📥 安装阶段

#### 使用自动化脚本（推荐）

```batch
# 右键 install.bat → 以管理员身份运行
.\install.bat
```

脚本会自动完成：
1. ✅ 检查测试模式
2. ✅ 停止 Sandboxie 服务
3. ✅ 备份原始文件
4. ✅ 复制新文件
5. ✅ 启动服务

#### 手动安装

```powershell
# 1. 启用测试模式（如未启用）
bcdedit /set testsigning on
shutdown /r /t 0

# 2. 停止服务
sc stop SbieSvc

# 3. 备份原始文件
xcopy "C:\Program Files\Sandboxie-Plus" "C:\Program Files\Sandboxie-Plus.backup" /E /I

# 4. 复制新文件
copy Release_Package_*\*.* "C:\Program Files\Sandboxie-Plus\" /Y

# 5. 启动服务
sc start SbieSvc
```

## 🐛 故障排除

### 问题1: 编译失败

**症状**: Visual Studio 报错，无法编译

**解决方案**:
```
1. 检查 WDK 是否安装
2. 安装 WDK 扩展:
   C:\Program Files (x86)\Windows Kits\10\Vsix\VS2019\WDK.vsix
3. 重启 Visual Studio
4. 确保选择了 Release + x64 配置
```

### 问题2: 驱动加载失败

**症状**: 服务无法启动，错误代码 52

**解决方案**:
```powershell
# 检查测试模式
bcdedit /enum | findstr testsigning

# 如果显示 No，启用测试模式
bcdedit /set testsigning on
shutdown /r /t 0
```

### 问题3: 功能仍需证书

**症状**: 设置高级功能时仍提示需要证书

**解决方案**:
```
1. 确认代码修改正确
2. 确认编译成功
3. 确认新文件已复制到安装目录
4. 重启 Sandboxie 服务: sc stop SbieSvc && sc start SbieSvc
5. 如果还不行，重启电脑
```

### 问题4: 蓝屏 (BSOD)

**症状**: 系统蓝屏，提示驱动错误

**解决方案**:
```
1. 重启进入安全模式 (F8)
2. 运行 restore.bat 恢复原始版本
3. 或手动删除驱动: sc delete SbieDrv
4. 检查编译配置是否正确（必须是 Release + x64）
5. 重新编译并安装
```

### 问题5: 签名失败

**症状**: build_and_sign.bat 报告签名失败

**解决方案**:
```
1. 确认以管理员身份运行
2. 检查证书是否创建成功
3. 手动创建证书（参考"编译和签名指南.md"）
4. 或跳过签名，直接在测试模式下使用
```

💡 **更多问题**: 查看 `编译和签名指南.md` 的"常见问题"章节

## 🎯 核心修改说明

### 修改的文件
```
只需修改 1 个文件: core/drv/verify.c
```

### 修改的函数
```
KphValidateCertificate() - 证书验证函数
```

### 修改的原理
```
原始逻辑:
1. 读取 Certificate.dat 文件
2. 验证数字签名
3. 检查证书类型、级别、过期时间
4. 根据证书设置功能权限

修改后逻辑:
1. 直接设置最高权限
2. 跳过所有验证
3. 返回成功
```

### 解锁的功能
```c
Verify_CertInfo.type = eCertEternal;      // 永久证书
Verify_CertInfo.level = eCertMaxLevel;    // 最高级别
Verify_CertInfo.opt_sec = 1;              // 安全功能
Verify_CertInfo.opt_enc = 1;              // 加密功能
Verify_CertInfo.opt_net = 1;              // 网络功能
Verify_CertInfo.opt_desk = 1;             // 桌面功能
Verify_CertInfo.active = 1;               // 激活状态
```
