## 🔨 编译步骤

### 方法一：使用 Visual Studio GUI

#### 1. 打开项目
```
1. 启动 Visual Studio 2019
2. 打开 Sandboxie\Sandbox.sln
3. 如果提示 WDK 扩展，点击安装（位于 C:\Program Files (x86)\Windows Kits\10\Vsix\VS2019）
```

#### 2. 配置编译选项
```
1. 在顶部工具栏选择：
   - 配置：Release
   - 平台：x64（或 Win32）

2. 如果有多个 Windows SDK，右键解决方案 -> 重定目标解决方案 -> 选择 10.0.19041
```

#### 3. 编译顺序（重要！）

**对于 x64 版本：**
```
步骤1: 编译 LowLevel (Win32)
  - 右键 core\low\LowLevel 项目
  - 选择"仅用于项目" -> "仅生成 LowLevel"
  - 平台选择 Win32

步骤2: 切换到 x64 平台

步骤3: 编译整个解决方案
  - 菜单：生成 -> 生成解决方案
  - 或按 Ctrl+Shift+B
```

#### 4. 编译输出位置
```
编译完成后，文件位于：
- 驱动：Sandboxie\core\drv\x64\Release\SbieDrv.sys
- 服务：Sandboxie\core\svc\x64\Release\SbieSvc.exe
- DLL：Sandboxie\core\dll\x64\Release\SbieDll.dll
- 控制台：Sandboxie\apps\control\x64\Release\SbieCtrl.exe
- 启动器：Sandboxie\apps\start\x64\Release\Start.exe
```

### 方法二：使用命令行编译

创建编译脚本 `build.bat`：

```batch
@echo off
echo ========================================
echo Sandboxie 编译脚本
echo ========================================

REM 设置 Visual Studio 环境
call "C:\Program Files (x86)\Microsoft Visual Studio\2019\Community\VC\Auxiliary\Build\vcvars64.bat"

REM 进入项目目录
cd /d "%~dp0"

echo.
echo [1/3] 编译 LowLevel (Win32)...
msbuild Sandbox.sln /t:LowLevel /p:Configuration=Release /p:Platform=Win32

echo.
echo [2/3] 编译主项目 (x64)...
msbuild Sandbox.sln /t:Build /p:Configuration=Release /p:Platform=x64

echo.
echo [3/3] 编译完成！
echo.
echo 输出文件位置：
echo - 驱动：core\drv\x64\Release\SbieDrv.sys
echo - 服务：core\svc\x64\Release\SbieSvc.exe
echo - DLL：core\dll\x64\Release\SbieDll.dll
echo.
pause
```

运行：
```powershell
.\build.bat
```

## 🔐 驱动签名

Windows 要求所有内核驱动必须签名。有三种方法：

### 方法一：测试模式（推荐用于个人使用）

#### 1. 启用测试模式

```powershell
# 以管理员身份运行 PowerShell

# 启用测试模式
bcdedit /set testsigning on

# 重启电脑
shutdown /r /t 0
```

重启后，桌面右下角会显示"测试模式"水印。

#### 2. 创建测试证书

```powershell
# 以管理员身份运行

# 创建测试证书
$cert = New-SelfSignedCertificate `
    -Type CodeSigningCert `
    -Subject "CN=Sandboxie Test Certificate" `
    -KeyUsage DigitalSignature `
    -FriendlyName "Sandboxie Test Cert" `
    -CertStoreLocation "Cert:\CurrentUser\My" `
    -TextExtension @("2.5.29.37={text}1.3.6.1.5.5.7.3.3", "2.5.29.19={text}")

# 导出证书
$password = ConvertTo-SecureString -String "YourPassword123" -Force -AsPlainText
Export-PfxCertificate -Cert $cert -FilePath "C:\SandboxieTestCert.pfx" -Password $password

# 导出公钥（用于安装到受信任的根证书）
Export-Certificate -Cert $cert -FilePath "C:\SandboxieTestCert.cer"
```

#### 3. 安装证书到受信任的根

```powershell
# 导入到受信任的根证书颁发机构
Import-Certificate -FilePath "C:\SandboxieTestCert.cer" -CertStoreLocation "Cert:\LocalMachine\Root"

# 导入到受信任的发布者
Import-Certificate -FilePath "C:\SandboxieTestCert.cer" -CertStoreLocation "Cert:\LocalMachine\TrustedPublisher"
```

#### 4. 签名驱动文件

创建签名脚本 `sign_driver.bat`：

```batch
@echo off
echo ========================================
echo Sandboxie 驱动签名脚本
echo ========================================

REM 设置路径
set DRIVER_PATH=core\drv\x64\Release\SbieDrv.sys
set CERT_PATH=C:\SandboxieTestCert.pfx
set CERT_PASSWORD=YourPassword123

REM 查找 signtool.exe
set SIGNTOOL="C:\Program Files (x86)\Windows Kits\10\bin\10.0.19041.0\x64\signtool.exe"

echo.
echo 正在签名驱动文件...
%SIGNTOOL% sign /f "%CERT_PATH%" /p "%CERT_PASSWORD%" /v /t http://timestamp.digicert.com "%DRIVER_PATH%"

if %ERRORLEVEL% EQU 0 (
    echo.
    echo ✅ 签名成功！
    echo.
    echo 验证签名...
    %SIGNTOOL% verify /v /pa "%DRIVER_PATH%"
) else (
    echo.
    echo ❌ 签名失败！
)

pause
```

运行：
```powershell
.\sign_driver.bat
```

#### 5. 签名所有可执行文件

创建 `sign_all.bat`：

```batch
@echo off
setlocal enabledelayedexpansion

set CERT_PATH=C:\SandboxieTestCert.pfx
set CERT_PASSWORD=YourPassword123
set SIGNTOOL="C:\Program Files (x86)\Windows Kits\10\bin\10.0.19041.0\x64\signtool.exe"

echo ========================================
echo 签名所有 Sandboxie 文件
echo ========================================

REM 定义需要签名的文件
set FILES=(
    "core\drv\x64\Release\SbieDrv.sys"
    "core\svc\x64\Release\SbieSvc.exe"
    "core\dll\x64\Release\SbieDll.dll"
    "apps\control\x64\Release\SbieCtrl.exe"
    "apps\start\x64\Release\Start.exe"
    "apps\ini\x64\Release\SbieIni.exe"
    "install\kmdutil\x64\Release\KmdUtil.exe"
    "SboxHostDll\x64\Release\SboxHostDll.dll"
    "msgs\x64\Release\SboxMsg.dll"
)

for %%F in %FILES% do (
    if exist "%%~F" (
        echo.
        echo 签名: %%~F
        %SIGNTOOL% sign /f "%CERT_PATH%" /p "%CERT_PASSWORD%" /v /t http://timestamp.digicert.com "%%~F"
    ) else (
        echo ⚠️  文件不存在: %%~F
    )
)

echo.
echo ========================================
echo 签名完成！
echo ========================================
pause
```

### 方法二：使用现有的代码签名证书

如果你有商业代码签名证书（如 DigiCert、Sectigo 等）：

```batch
@echo off
REM 使用商业证书签名

set SIGNTOOL="C:\Program Files (x86)\Windows Kits\10\bin\10.0.19041.0\x64\signtool.exe"

REM 方式1：使用证书存储中的证书
%SIGNTOOL% sign /n "Your Certificate Name" /t http://timestamp.digicert.com /fd sha256 /v SbieDrv.sys

REM 方式2：使用 PFX 文件
%SIGNTOOL% sign /f "YourCert.pfx" /p "password" /t http://timestamp.digicert.com /fd sha256 /v SbieDrv.sys

REM 方式3：使用硬件令牌（USB Key）
%SIGNTOOL% sign /sha1 "证书指纹" /t http://timestamp.digicert.com /fd sha256 /v SbieDrv.sys
```

## 🚨 常见问题

### 问题1：编译错误 - 找不到 WDK

**解决方案：**
```
1. 确认 WDK 已安装：C:\Program Files (x86)\Windows Kits\10\
2. 安装 WDK 扩展：C:\Program Files (x86)\Windows Kits\10\Vsix\VS2019\WDK.vsix
3. 重启 Visual Studio
```

### 问题2：驱动加载失败 - 代码 52

**解决方案：**
```powershell
# 检查测试模式是否启用
bcdedit /enum | findstr testsigning

# 如果显示 No，启用测试模式
bcdedit /set testsigning on
shutdown /r /t 0
```

### 问题3：签名验证失败

**解决方案：**
```powershell
# 确保证书已安装到正确的存储
certmgr.msc

# 检查：
# - 受信任的根证书颁发机构
# - 受信任的发布者
```

### 问题4：蓝屏 (BSOD)

**解决方案：**
```
1. 进入安全模式
2. 删除驱动：sc delete SbieDrv
3. 恢复原始文件
4. 检查编译配置是否正确（Release, x64）
```

## 📝 完整自动化脚本

创建 `build_and_sign.bat`：

```batch
@echo off
setlocal enabledelayedexpansion

echo ========================================
echo Sandboxie 完整编译和签名脚本
echo ========================================

REM 配置
set VS_PATH=C:\Program Files (x86)\Microsoft Visual Studio\2019\Community
set WDK_PATH=C:\Program Files (x86)\Windows Kits\10
set CERT_PATH=C:\SandboxieTestCert.pfx
set CERT_PASSWORD=YourPassword123

echo.
echo [1/5] 设置编译环境...
call "%VS_PATH%\VC\Auxiliary\Build\vcvars64.bat"

echo.
echo [2/5] 编译 LowLevel (Win32)...
msbuild Sandbox.sln /t:LowLevel /p:Configuration=Release /p:Platform=Win32 /m

echo.
echo [3/5] 编译主项目 (x64)...
msbuild Sandbox.sln /t:Build /p:Configuration=Release /p:Platform=x64 /m

echo.
echo [4/5] 签名所有文件...
set SIGNTOOL="%WDK_PATH%\bin\10.0.19041.0\x64\signtool.exe"

for %%F in (
    "core\drv\x64\Release\SbieDrv.sys"
    "core\svc\x64\Release\SbieSvc.exe"
    "core\dll\x64\Release\SbieDll.dll"
    "apps\control\x64\Release\SbieCtrl.exe"
    "apps\start\x64\Release\Start.exe"
) do (
    if exist "%%~F" (
        echo 签名: %%~F
        %SIGNTOOL% sign /f "%CERT_PATH%" /p "%CERT_PASSWORD%" /t http://timestamp.digicert.com /v "%%~F" >nul 2>&1
        if !ERRORLEVEL! EQU 0 (
            echo   ✅ 成功
        ) else (
            echo   ❌ 失败
        )
    )
)

echo.
echo [5/5] 创建发布包...
set OUTPUT_DIR=Release_Package
if exist "%OUTPUT_DIR%" rmdir /s /q "%OUTPUT_DIR%"
mkdir "%OUTPUT_DIR%"

xcopy "core\drv\x64\Release\SbieDrv.sys" "%OUTPUT_DIR%\" /Y
xcopy "core\svc\x64\Release\SbieSvc.exe" "%OUTPUT_DIR%\" /Y
xcopy "core\dll\x64\Release\SbieDll.dll" "%OUTPUT_DIR%\" /Y
xcopy "apps\control\x64\Release\SbieCtrl.exe" "%OUTPUT_DIR%\" /Y
xcopy "apps\start\x64\Release\Start.exe" "%OUTPUT_DIR%\" /Y
xcopy "apps\ini\x64\Release\SbieIni.exe" "%OUTPUT_DIR%\" /Y
xcopy "install\kmdutil\x64\Release\KmdUtil.exe" "%OUTPUT_DIR%\" /Y

echo.
echo ========================================
echo ✅ 完成！
echo ========================================
echo.
echo 输出目录: %OUTPUT_DIR%
echo.
echo 下一步：
echo 1. 确保测试模式已启用: bcdedit /set testsigning on
echo 2. 停止现有服务: sc stop SbieSvc
echo 3. 复制文件到安装目录
echo 4. 启动服务: sc start SbieSvc
echo.
pause
```

## ✅ 验证解锁成功

启动 Sandboxie 后，检查以下功能是否可用：

1. **打开 Sandboxie Control**
2. **右键沙盒 -> 沙盒设置**
3. **检查以下选项是否可用（不显示需要证书的提示）：**
   - ✅ 安全 -> 使用安全模式 (UseSecurityMode)
   - ✅ 安全 -> 系统调用锁定 (SysCallLockDown)
   - ✅ 资源访问 -> 加密沙盒 (ConfidentialBox)
   - ✅ 网络 -> DNS 过滤 (NetworkDnsFilter)
   - ✅ 外观 -> 使用独立桌面 (UseSandboxDesktop)

如果这些选项都可以正常设置，说明解锁成功！
